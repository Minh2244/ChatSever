Trong hệ thống, MySQL Server được sử dụng để lưu trữ các dữ liệu nhẹ như thông tin người dùng, nội dung tin nhắn và đặc biệt là các đường link (URL) của hình ảnh, video và file. Các dữ liệu nặng như ảnh, video hoặc tài liệu không được lưu trực tiếp vào server mà được upload lên Firebase Storage. Khi người dùng gửi file, frontend sẽ upload trực tiếp file lên Firebase và nhận về một đường link. Sau đó, client gửi đường link này đến server thông qua WebSocket, server lưu lại vào MySQL Server và gửi cho các client khác. Khi nhận được link, phía client sẽ hiển thị nội dung thông qua các thẻ HTML như <img> hoặc <video>. Cách làm này giúp giảm tải cho server, tối ưu hiệu năng và đảm bảo hệ thống hoạt động ổn định khi dữ liệu tăng lên.

Cách hoạt động chi tiết
Khi người dùng thực hiện gửi một file như hình ảnh, video hoặc tài liệu, hệ thống sẽ xử lý theo một quy trình tối ưu nhằm giảm tải cho server. Đầu tiên, phía client (frontend) sẽ trực tiếp upload file lên Firebase Storage thay vì gửi qua server Java. Sau khi upload hoàn tất, Firebase sẽ trả về một đường link (URL) đại diện cho file đó. Tiếp theo, client sẽ gửi đường link này đến server thông qua WebSocket dưới dạng một tin nhắn văn bản kèm thông tin người gửi và người nhận.
Server Java sau khi nhận được dữ liệu sẽ tiến hành lưu đường link này vào MySQL Server như một phần của lịch sử tin nhắn, sau đó phân phối lại cho các client liên quan (gửi cho tất cả hoặc gửi riêng tùy trường hợp). Khi client nhận được tin nhắn chứa đường link, giao diện web sẽ tự động hiển thị nội dung bằng cách nhúng link vào các thẻ HTML tương ứng như <img> đối với ảnh hoặc <video> đối với video.
Toàn bộ quá trình này giúp server chỉ xử lý dữ liệu nhẹ (text và URL), trong khi các file dung lượng lớn được xử lý bởi hệ thống lưu trữ chuyên dụng. Nhờ đó, hệ thống đảm bảo hiệu năng cao, giảm tải cho backend và vẫn đáp ứng tốt nhu cầu truyền tải dữ liệu đa phương tiện của người dùng.
