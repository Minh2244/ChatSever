CHATSERVER PROJECT REPORTI. PROJECT MOTIVATIONOriginally a classroom assignment using Java Socket and Java Swing, the initial version faced significant issues with UI aesthetics, low security, and data corruption (binary encoding errors and excessive storage consumption). This project was upgraded to a modern Web-based architecture to enhance user experience, ensure data integrity, and enable global accessibility.II. KEY FEATURESUser Management: Registration, authentication, and real-time online status tracking.Messaging Modes: Public global chat and private one-on-one messaging.Multimedia Interaction: Support for multi-format file sharing, images, videos, emojis, and GIFs.Message Forwarding: Capability to share messages and media with other users within the chat history.Real-time Communication: Integrated voice and video calling features via web protocols.III. TECH STACKBackend: Java Spring Boot, WebSocket, Docker.Frontend: React, Vite.Database & Storage: MySQL Server, Firebase Storage, Cloudinary.Networking: WebRTC, TURN/STUN Services (Metered).Deployment: Render, Vercel, Railway.IV. RESOURCE OPTIMIZATIONTo ensure stable performance on a low-spec server (500MB RAM), the following optimizations were implemented:Client-side Upload: Heavy files are uploaded directly to Cloud Storage, bypassing the backend to prevent memory overflow.Metadata Management: The server only processes and stores URLs, significantly reducing the workload on the Java Virtual Machine.Multi-threading: Efficient handling of concurrent WebSocket connections to maintain system stability under load.
